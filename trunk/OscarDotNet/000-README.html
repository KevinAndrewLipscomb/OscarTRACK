000-README


Abstract

This document describes additional steps that you must perform, after checking out the OscarTRACK project from its repository, to achieve a clean compilation or to execute the application. This document does not cover how to meet other prerequisites, such as setting up an ASP.NET server, creating and populating the database schema, and setting up a cron daemon.

1 Recommended methods

The use of NTFS junction points is recommended in this document because it allows Subversion operations to take place without undesirable impact to project environments, and without violating Subversion rules. If you simply exported OscarTRACK from its repository and you do not intend to perform subsequent updates or any other Subversion operations on it (not recommended), you can simply make copies instead of NTFS junction points.

2 Establish hooks to adjunct projects

2.1 Hook up the KiUsercontrolFoundry project

OscarTRACK uses UserControls from the KiUsercontrolFoundry adjunct project. The KiUsercontrolFoundry is used by multiple production-eligible projects and is version controlled independently from those production-eligible projects. To hook up the KiUsercontrolFoundry project, do the following:

  1. Check out the https://kveo-it-project.svn.sourceforge.net/svnroot/kveo-it-project/trunk/foundry/usercontrol/ki project. Put it in a KiUsercontrolFoundry folder. Make sure the KiUsercontrolFoundry folder is not inside the OscarTRACK folder tree.
  2. Create a junction point named ki inside the OscarTRACK/usercontrol folder that points to the KiUsercontrolFoundry/usercontrol/ki folder. You should end up with two folders inside the OscarTRACK/usercontrol folder: app and ki. You can make junction points with Michael Elsdoerfer’s NTFS Link tool or, on Vista-based systems (including Windows 7, Windows Server 2008, and later systems), the prototype commands would be:

    cd OscarTRACK/usercontrol
    mklink /j ki KiUserControlFoundry/usercontrol/ki

2.2 Hook up the KiDotnetFoundation project (only if compiling)

OscarTRACK uses object code from the KiDotnetFoundation adjunct project. The KiDotnetFoundation is used by multiple production-eligible projects and its source code is version controlled independently from those production-eligible projects. The object code is already provided as OscarTRACK/bin/ki.dll to enable execution of OscarTRACK in a production environment, but to hook up the KiDotnetFoundation project for the sake of a clean compile of OscarTRACK, do the following:

  1. Check out the https://kveo-it-project.svn.sourceforge.net/svnroot/kveo-it-project/trunk/foundation/dotnet/ki project. Put it in a KiDotnetFoundation folder. Make sure the KiDotnetFoundation folder is not inside the OscarTRACK folder tree.
  2. Create a junction point named ki inside the OscarTRACK/lib folder that points to the KiDotnetFoundation folder. You should end up with a folder inside the OscarTRACK/lib folder called ki. You can make junction points with Michael Elsdoerfer’s NTFS Link tool or, on Vista-based systems (including Windows 7, Windows Server 2008, and later systems), the prototype commands would be:

    cd OscarTRACK/lib
    mklink /j ki KiDotnetFoundation

3 Put appropriate config files in the application root folder

OscarTRACK will not run without Web.config and unpublished.config files in its top-level folder. Because the contents of those files are environment-specific, and we wish to keep the canonical ASP.NET parts of the OscarTRACK folder tree environment-neutral, those config files are kept in a non-ASP.NET-canonical OscarTRACK/config subfolder tree. To put config files that are appropriate for your environment into the top-level OscarTRACK folder, do the following:

  1. If there is no unpublished.config file inside the OscarTRACK/config folder tree (this should be true for projects kept in public repositories), copy the OscarTRACK/config/template-unpublished.config-template file to an appropriate subfolder of OscarTRACK/config, rename it to unpublished.config, and revise its contents appropriately. Do not add any unpublished.config file to any public repository.
  2. Copy the appropriate unpublished.config file from within the OscarTRACK/config folder tree to the OscarTRACK top-level folder.
  3. Copy the appropriate Web.config file from within the OscarTRACK/config folder tree to the OscarTRACK top-level folder.

From the Windows command line, the prototype commands would be:

cd OscarTRACK
copy config/{appropriate-path}/unpublished.config .
copy config/{appropriate-path}/Web.config .

4 Set up cron jobs

Some FromPaper2Web applications include features that support scheduled unattended jobs, such as notifications and report generation, data scrubs, and other periodic tasks. Crontab entries for any such jobs that are part of OscarTRACK are located in noninteractive/.crontab.
